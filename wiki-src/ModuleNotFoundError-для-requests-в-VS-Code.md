# Симптомы

После установки библиотеки requests программа падает с ошибкой:

`ModuleNotFoundError: No module named 'requests'`

## Почему это происходит

В VS Code интерпретатор `Python` и команда `pip` могут указывать на разные версии Python.

### Типичная ситуация:

- `pip install requests` выполнен для одного Python

- VS Code запускает код другим Python

- результат — модуль «не найден»

### Почему виртуальное окружение решает проблему

Виртуальное окружение (venv):

- жёстко связывает:

>  - интерпретатор Python

> - установленные библиотеки

- исключает конфликт версий

- гарантирует, что `pip` и `python` работают вместе

# Правильный алгоритм решения
## 1️⃣ Проверить количество Python

Если их несколько — рекомендуется оставить один.

## 2️⃣ Создать виртуальное окружение
```bash
python -m venv venv
```

## 3️⃣ Активировать окружение

### Автоматически:

VS Code предложит использовать новое окружение → Yes

### Вручную:

```bash
.\venv\Scripts\activate
```

## 4️⃣ Выбрать интерпретатор

- Ctrl + Shift + P

- Python: Select Interpreter

- выбрать Python из venv

## 5️⃣ Перезапустить VS Code

Закрыть и снова открыть IDE.

## 6️⃣ Проверить, что окружение активно

В терминале должен появиться префикс:

`(venv)`

## 7️⃣ Установить requests
```bash
pip install requests
```

### Частые ошибки учеников

- ❌ Устанавливать requests до выбора интерпретатора
- ❌ Игнорировать предложение VS Code выбрать окружение
- ❌ Работать без venv при нескольких Python

### Итог

Если `pip install` прошёл успешно, но `import` не работает —
проблема почти всегда в интерпретаторе Python.

**Решение:** виртуальное окружение + правильный интерпретатор.
